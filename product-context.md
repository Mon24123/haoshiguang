# 好时光产品上下文交接文档

## 一句话定位

好时光是一个桌面常驻时间记录挂件，核心目标是降低“记录自己正在做什么”的启动成本，让用户随时看到当前事项和已用时间。

## 当前产品形态

- Electron 桌面应用。
- 主界面是完整面板，包含计时、任务、待办池、分类统计、时间记录。
- 桌面上有一个小卡片挂件，显示当前任务和计时。
- 小卡片可展开控制区，也可以收起成屏幕边缘的小胶囊。
- 数据存储在 Electron 本地 userData 中，目前没有云同步。

## 核心用户场景

1. 用户突然开始做一件事，希望 3 秒内记下来。
2. 用户不想打开飞书/表格/大型 App，只想快速开始计时。
3. 用户一天内频繁切换任务，希望回头知道自己到底做了什么。
4. 用户希望同一个任务可以分多段记录，而不是生成一堆重复记录。
5. 用户希望晚上可以复制日报，快速总结当天时间流向。

## 核心功能

- 当前事项计时：开始、暂停、结束。
- 桌面卡片：常驻显示当前任务、当前时长。
- 快速切换：从桌面卡片切换已有任务或输入新任务。
- 今日任务：当天要完成的任务，可勾选完成。
- 待办池：不限定具体日期的任务，可设置启动日期或立即开始。
- 每日固定任务：每天自动生成的固定任务模板。
- 分类：任务/记录可归类，例如工作、项目、生活、健康、其他。
- 时间记录：同一任务的多段时间合并展示，可修改时间、备注、删除。
- 分类统计：按日/周/月查看分类投入。
- 复制日报：将当天任务和时间记录复制成文字。

## 产品原则

- 记录入口必须比打开飞书更快。
- 桌面卡片要有存在感，但不能压迫用户。
- 能继续做同一个任务，就不要重复创建同名任务。
- 分类要支持用户自定义，不要替用户规定人生模块。
- 功能可以多，但默认界面要尽量少打扰。
- 时间记录必须可纠错，因为用户一定会忘记开始、忘记结束。

## 当前最重要的体验问题

- 分类选择菜单有时仍可能因为刷新或 hover 边界产生卡顿。
- 桌面卡片在浏览器系统级全屏时可能不可见，这是 macOS/Windows 空间层级问题。
- Windows 用户需要明确的退出/收起入口，否则会不知道怎么关。
- 自动更新还没有做，新版本需要重新发安装包。
- 统计视图还在探索中，当前“分类泡泡”比环形图更适合继续迭代。

## 技术结构

- `main.js`：Electron 主进程，负责窗口创建、IPC、打包后的桌面行为。
- `preload.js`：暴露安全 IPC API 给页面。
- `index.html`：完整面板页面结构。
- `script.js`：完整面板业务逻辑、数据存储、渲染、计时。
- `styles.css`：完整面板样式。
- `bubble.html`：桌面卡片结构。
- `bubble.js`：桌面卡片交互、计时状态同步。
- `bubble.css`：桌面卡片样式。
- `package.json`：启动和打包脚本。

## 打包命令

```bash
npm run package:mac
npm run package:win
```

## 给下一个 AI 的提醒

- 不要先大改架构，先复现用户描述的问题。
- 修改分类菜单、时间记录 hover、桌面卡片窗口尺寸时，要同时看 `script.js`、`styles.css`、`main.js`、`bubble.js`。
- 这个项目没有前端框架，所有状态和渲染都在原生 JS 里，频繁 `render()` 容易把用户正在编辑的菜单刷掉。
- 用户非常在意“点了有没有立刻反馈”，所以删除、保存、切换分类要优先做即时 UI 更新。
- 用户审美偏向干净、有活人味、低 AI 味，不喜欢过度花哨、死板黑白、老土配色。
