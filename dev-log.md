# 好时光开发日志与未来计划

## 2026-05-20 至 2026-05-23 阶段总结

这个阶段完成了从网页 Demo 到 Electron 桌面应用的 MVP。产品从“一个计时网页”逐渐演化为“桌面常驻时间记录挂件”。

## 已完成

- 创建 Electron 应用。
- 完成 macOS 桌面卡片。
- 完成 Windows 打包。
- 应用名称改为“好时光”。
- 桌面卡片支持拖动、展开、收起。
- 桌面卡片支持暂停、结束、快速备注。
- 桌面卡片支持快速切换任务。
- 完整面板支持日期切换。
- 今日任务支持添加、分类、开始、继续、勾选、删除。
- 同名或同任务多段时间合并展示。
- 时间记录支持改时间、写备注、删除。
- 删除时间记录前加入确认。
- 支持补记任务。
- 支持待办池。
- 支持每日固定任务。
- 支持分类新增、排序、改名、删除。
- 支持主题色选择。
- 支持复制日报。
- 新增 Windows 用户退出和收起体验。

## 最近一次修改重点

- 桌面卡片右上角 `×` 改为收起到屏幕边缘小胶囊。
- Windows 关闭窗口时改为退出应用。
- 优化分类菜单、每日任务、待办池、时间记录编辑时被定时刷新打断的问题。
- 优化复制日报文案格式。
- 新增内测核心说明和交接文档。

## 仍需重点验证

1. 桌面卡片收起后，点小胶囊是否能稳定恢复。
2. Windows 用户是否能理解“收起”和“退出”的区别。
3. 今日任务里修改分类是否仍有卡顿。
4. 时间记录 hover 小窗口是否稳定，不会莫名弹出或消失。
5. 修改时间段保存后，是否留在原位置，且记录确实更新。
6. 复制日报格式是否适合直接发给朋友或贴到飞书。
7. 异常退出后，正在计时的任务是否能恢复。

## 已知问题

- 没有自动更新，版本升级仍需要重新发送安装包。
- 浏览器全屏时，桌面卡片可能被系统空间遮挡。
- 数据没有云同步和跨设备同步。
- 分类统计的视觉形式还没有完全定稿。
- 原生 JS 手写渲染较多，局部交互容易被全局 render 影响。

## 优先级较高的下一步

1. 梳理渲染逻辑，减少全量 `render()` 对编辑状态的干扰。
2. 给 Windows 用户补一个更明显的退出入口，或做托盘菜单。
3. 增加数据导出/备份入口，降低内测数据丢失风险。
4. 做一份内测反馈表，收集朋友实际使用问题。
5. 设计自动更新方案。

## 未来功能池

- 自动更新。
- 托盘菜单。
- 快捷键快速开始/暂停/结束。
- 更漂亮的日报图片生成。
- 日/周/月统计图。
- 任务备注词云。
- 飞书/原子笔记同步。
- 数据备份和恢复。
- 用户自定义桌面卡片形象。
- 新手引导和示例数据。

## 给后续开发的建议

- 每次只改一个明确问题，改完马上验证。
- 尽量不要同时改视觉、数据结构、交互逻辑。
- 涉及计时、记录、日期切换的改动要格外小心。
- 如果要继续扩大功能，建议先把状态管理拆清楚，否则后续会越来越难改。
